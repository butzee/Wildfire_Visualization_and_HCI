Task

This project was developed from the task to create an interactive visualization application, which originated from the Lecture "Visualization and HCI".

Therefore, a more complex application was required, stretching beyond static illustrations and simple graph, which is why this program enables highly interactive visualizations of the underlying data, including customizable scaling, playback speed and filtering options.

The design, project idea and user experience orignated from the Five Design-Sheet methodology, which was mandatory in order to pass the module.

Techniques

To enable a seamless user experience, the JavaScript library Maptalks was used to illustrate the map and allow the robust rendering of a large quantity of data.

The application also utilizes special DeckGL plugin for Maptalks, which makes the rendering of thousand of datapoints per second possible.

Only this, and a prefetching of exactly the data needed for the current visualization, brings the smooth illustration of 2.3 million wildfires to almost any computer.

Problems

The major problem of this project was the size of the underlying dataset, spanning 2.3 million database rows. A visualization of wildfires per year, each year being display for one second, would therefore require loading and rendering over 80.000 data points in under half a second.

If the whole thing would've taken longer, the end-user wouldn't be able to really observe any data.

Another problem was missing data for a big portion of the fires, which had to be either imputed or removed.
